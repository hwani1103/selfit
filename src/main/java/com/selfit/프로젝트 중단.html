여태까지 내가 활용한 Spring, Spring Boot, JPA 등의 기술을 최대한 주석으로 정리한 후 이번 프로젝트도 중단. 나중에 다시 참고할 일이 있을 수 있으니 최대한 정리해놓는다. 당분간은 Javascript, React, Node,JS를 공부해서 취업의 폭을 넓히고, 추후에 대비하도록 하자 일단 Selifit에서 구현한 기능은 다음과 같다. 1.회원가입 회원가입을 하면 아이디, 닉네임 중복확인(ajax) 비밀번호 두개가 서로 일치해야함.(javascript) 중복확인과 비밀번호가 정상 입력되면 이메일로 토큰값이 전송된다. 토큰값을 정상 입력하면 회원가입 & 인증 & 로그인까지 성공 토큰을 보내지 않았더라도 회원정보는 저장되고, 그상태에서 토큰 재전송 1회 가능하며 그 이후는 1시간에 1번씩만 가능. 언제든 토큰을 인증하면 인증된 회원으로 처리되고, 그 후부터 로그인이 가능함. 2.로그인 로그인은 당연히 이메일&비밀번호가 일치해야 로그인이 가능함. 이거는 일반적인 post요청으로 처리 비밀번호를 분실했을때에도 회원가입 인증때처럼 인증토큰을 보낼 수 있음. 인증회원이 비밀번호 분실로 인해 이메일 인증을 통한 로그인을 실시하는 경우 임시비밀번호를 토큰값으로 설정하고, 메인화면에 비밀번호를 변경하라는 안내문구를 띄워준다. 로그인 / 로그아웃 처리는 스프링 시큐리티가 일임한다. MemberService쪽에 관련 메서드들이 있음. 3.프로필 설정 프로필에는 현재 로그인 회원의 정보와 해당 프로필 회원의 정보가 다 있어야 함. 그 프로필의 주인에게만 프로필 수정 등의 옵션이 보여야 하기 때문. 프로필 수정에는 Cropper 라이브러리가 사용되었다.(사진 등록/자르기 등) 관심주제는 Tagify 라이브러리로 사용되었고 비교적 유연하게 구현했다고 생각함. 일단 이정도까지 구현했고 기술의 세세한 내용들은 직접 코드쪽에 주석으로 남김 html파일은 따로 추가 정리하진 않았다. 라이브러리 사용법들이나 부트스트랩 문법등을 자세히 알면 될듯. 타임리프 문법도 중요하지만.. 다시 사용해야 한다면 그 때 한번 더 공부한다고 생각해야 할듯. 아마 대기업에 취업해서 다시 Java를 써야하지 않는이상, 다시 자바 스프링 타임리프의 조합을 사용할 일이 많지는 않을듯함.
